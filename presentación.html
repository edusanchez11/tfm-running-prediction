Predicción del Rendimiento en Running

Un enfoque aplicado con Machine Learning

Eduardo Sánchez


Objetivo: Desarrollar un sistema que predice el rendimiento en corredores, desde el análisis de datos de Strava hasta una app funcional.

El Problema y la Solución

El Problema (El "Por Qué")

  • La planificación del running se basa a menudo en la experiencia subjetiva.
  • Faltan herramientas científicas precisas para predecir y optimizar.

¿Qué hace el sistema? (Funcionalidad)

  • Ofrece predicciones personalizadas del rendimiento (pace).
  • Genera planes de entrenamiento adaptativos.
  • Simula la progresión futura del corredor.

¿Para Qué? (Casos de Uso)

  • Para Corredores: Optimizar su entrenamiento, prevenir lesiones y establecer objetivos realistas.
  • Para Entrenadores: Usar una herramienta científica para diseñar planes personalizados y monitorizar a sus atletas.

Resultados Clave del Modelo

¿Qué tan bueno es el modelo?

  • Precisión: Predice el pace con un error de solo 29 segundos.
  • Fiabilidad (R²): Explica el 78.5% del rendimiento.
  • Modelo: El modelo final es un Random Forest por su gran equilibrio.

¿Qué predice tu rendimiento?

  • Eficiencia Aeróbica (68%): Es el factor MÁS importante.
  • Volumen Semanal (14%): Cuántos km corres a la semana.
  • Consistencia (9%): La regularidad de tus ritmos y entrenamientos.

El Resultado: MVP con Streamlit

La Aplicación Web

  • Se desarrolló un Producto Mínimo Viable (MVP) como una app web interactiva.
  • Tecnología: Construida en Streamlit (Python).
  • ¿Por qué Streamlit? Permite un desarrollo rápido y una integración nativa con los modelos de Machine Learning.

Funcionalidad (Demo)

  • Carga automática del CSV de Strava.
  • Dashboard de estadísticas y KPIs.
  • Simulador de Progresión ML.
  • Generador de Planes adaptativo (según nivel y objetivo).
  • Interfaz con el diseño y colores de Strava.
